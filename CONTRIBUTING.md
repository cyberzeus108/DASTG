# Contributing to D.A.S.T.G.

Thank you for your interest in contributing to D.A.S.T.G.! 

## Guidelines
- Use the folder structure as defined.
- All design files must include a short description in a `README.md` in their folder.
- Issues and pull requests should be descriptive and linked to the corresponding module.
- Maintain consistent naming for files: lowercase, hyphens for spaces, avoid special characters.

## Code of Conduct
Be respectful, collaborative, and professional.
